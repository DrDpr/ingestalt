## Scroll Animation Implementation

Using Intersection Observer API:

```typescript
const useScrollAnimation = () => {
  const ref = useRef<HTMLDivElement>(null);
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => setIsVisible(entry.isIntersecting),
      { threshold: 0.2 }
    );
    
    if (ref.current) observer.observe(ref.current);
    return () => observer.disconnect();
  }, []);

  return { ref, isVisible };
};
```

function DemoCanvas() {
  const canvasRef = useRef<HTMLDivElement>(null);
  const [nodes, setNodes] = useState([
    { id: '1', x: 20, y: 20, label: 'API Gateway', color: '#3b82f6', type: 'api' },
    { id: '2', x: 50, y: 20, label: 'User Service', color: '#8b5cf6', type: 'api' },
    { id: '3', x: 80, y: 20, label: 'Auth Service', color: '#ec4899', type: 'api' },
    { id: '4', x: 35, y: 50, label: 'User DB', color: '#8b5cf6', type: 'database' },
    { id: '5', x: 65, y: 50, label: 'Session DB', color: '#ec4899', type: 'database' },
    { id: '6', x: 50, y: 75, label: 'Dashboard', color: '#3b82f6', type: 'frontend' },
  ]);
  const [dragging, setDragging] = useState<string | null>(null);
  const [offset, setOffset] = useState({ x: 0, y: 0 });

  const handleMouseDown = (e: React.MouseEvent, nodeId: string) => {
    const node = nodes.find(n => n.id === nodeId);
    if (!node || !canvasRef.current) return;

    const rect = canvasRef.current.getBoundingClientRect();
    const nodeX = (node.x / 100) * rect.width;
    const nodeY = (node.y / 100) * rect.height;

    setDragging(nodeId);
    setOffset({
      x: e.clientX - nodeX,
      y: e.clientY - nodeY
    });
  };

  const handleMouseMove = (e: React.MouseEvent) => {
    if (!dragging || !canvasRef.current) return;

    const rect = canvasRef.current.getBoundingClientRect();
    const x = ((e.clientX - offset.x - rect.left) / rect.width) * 100;
    const y = ((e.clientY - offset.y - rect.top) / rect.height) * 100;

    setNodes(prev => prev.map(node =>
      node.id === dragging
        ? { ...node, x: Math.max(5, Math.min(95, x)), y: Math.max(5, Math.min(95, y)) }
        : node
    ));
  };

  const handleMouseUp = () => {
    setDragging(null);
  };

  return (
    <div
      ref={canvasRef}
      className="relative w-full h-full bg-gradient-to-br from-background via-card to-background"
      onMouseMove={handleMouseMove}
      onMouseUp={handleMouseUp}
      onMouseLeave={handleMouseUp}
    >
      {/* Grid background */}
      <div 
        className="absolute inset-0 opacity-10"
        style={{
          backgroundImage: `
            linear-gradient(to right, oklch(1 0 0 / 0.1) 1px, transparent 1px),
            linear-gradient(to bottom, oklch(1 0 0 / 0.1) 1px, transparent 1px)
          `,
          backgroundSize: '40px 40px'
        }}
      />

      {/* Connection lines */}
      <svg className="absolute inset-0 w-full h-full pointer-events-none">
        <line x1="20%" y1="20%" x2="35%" y2="50%" stroke="#3b82f6" strokeWidth="2" opacity="0.3" />
        <line x1="50%" y1="20%" x2="35%" y2="50%" stroke="#8b5cf6" strokeWidth="2" opacity="0.3" />
        <line x1="50%" y1="20%" x2="65%" y2="50%" stroke="#8b5cf6" strokeWidth="2" opacity="0.3" />
        <line x1="80%" y1="20%" x2="65%" y2="50%" stroke="#ec4899" strokeWidth="2" opacity="0.3" />
        <line x1="35%" y1="50%" x2="50%" y2="75%" stroke="#3b82f6" strokeWidth="2" opacity="0.3" />
        <line x1="65%" y1="50%" x2="50%" y2="75%" stroke="#ec4899" strokeWidth="2" opacity="0.3" />
      </svg>

      {/* Nodes */}
      {nodes.map(node => (
        <div
          key={node.id}
          className="absolute cursor-move transition-transform hover:scale-110"
          style={{
            left: `${node.x}%`,
            top: `${node.y}%`,
            transform: 'translate(-50%, -50%)',
          }}
          onMouseDown={(e) => handleMouseDown(e, node.id)}
        >
          <div 
            className="px-6 py-4 rounded-2xl border-2 backdrop-blur-xl shadow-2xl transition-all duration-300 min-w-[160px]"
            style={{
              backgroundColor: `${node.color}15`,

              boxShadow: dragging === node.id ? `0 0 30px ${node.color}60` : `0 0 15px ${node.color}30`
            }}
          >
            <div className="text-sm font-bold text-center" style={{ color: node.color }}>
              {node.label}

            <div className="text-xs text-muted-foreground text-center mt-1 capitalize">

            </div>

<content># Ingestalt Landing Page

A modern, premium landing page design that visually aligns with Ingestalt's node-based canvas interface.

## Overview

The landing page embodies Ingestalt's core philosophy: **"Seeing something as a whole rather than the sum of its parts."**

It features:
- Immersive animated node graph background
- Smooth scroll-based animations
- Interactive canvas preview
- Minimal, intelligent design aesthetic
- Full responsive support

## Page Structure

### Route
- **URL**: `/landing`
- **File**: `app/landing/page.tsx`

### Components

#### 1. AnimatedNodeGraph (`components/AnimatedNodeGraph.tsx`)
- **Purpose**: Ambient background animation with floating nodes
- **Features**:
  - 8 floating nodes with unique animation paths
  - Animated connection lines between nodes
  - Grid overlay and radial gradient background
  - GPU-accelerated CSS animations (12-15s duration)
  - Ambient light effects with pulse animations

#### 2. HeroSection (`components/HeroSection.tsx`)
- **Purpose**: Main headline with staggered text reveal
- **Features**:
  - Large bold headline with gradient text effect
  - Staggered word-by-word fade-in animation (0.1s delays)
  - Two CTA buttons with hover effects
  - Scroll indicator with bounce animation
  - Responsive typography (6xl → 8xl → 9xl)

#### 3. FeaturesSection (`components/FeaturesSection.tsx`)
- **Purpose**: Showcase key product benefits
- **Features**:
  - 3-column grid layout (responsive)
  - Intersection Observer for scroll-triggered animations
  - Feature cards with hover lift and glow effects
  - Gradient backgrounds on hover
  - Icons: Network, Eye, HardDrive

**Features Highlighted**:
1. Visual Architecture - Transform complexity into clarity
2. Instant Understanding - See relationships at a glance
3. Local-First - Privacy-focused, no cloud dependencies

#### 4. InteractiveCanvasPreview (`components/InteractiveCanvasPreview.tsx`)
- **Purpose**: Live interactive demo of the canvas
- **Features**:
  - Fully draggable nodes (6 demo nodes)
  - Real-time connection lines
  - Tutorial overlay (auto-dismisses after 5s)
  - Grid background with gradient
  - Responsive container (600px → 800px height)
  - Node types: API, Database, Frontend

#### 5. CTASection (`components/CTASection.tsx`)
- **Purpose**: Final conversion point
- **Features**:
  - Large headline with gradient text
  - Two CTA buttons (Get Started, Learn More)
  - Ambient glow effects
  - Scroll-triggered fade-in
  - Footer info (hackathon details)

## Design System

### Typography
- **Headings**: Space Grotesk (--font-heading)
- **Body**: Geist Mono (--font-mono)
- **Sizes**: 
  - Hero: 6xl → 8xl → 9xl
  - Section headers: 4xl → 6xl → 7xl
  - Body: xl → 2xl → 3xl

### Color Palette
```css
--node-api: #3b82f6 (blue)
--node-database: #8b5cf6 (purple)
--node-frontend: #ec4899 (pink)
--glow-primary: rgba(59, 130, 246, 0.5)
```

### Animations

All animations are defined in `app/globals.css`:

#### Float Animations (8 variants)
- Duration: 10-15s
- Easing: ease-in-out
- Transform: translate + scale
- Staggered start delays (0.5s - 3.5s)

#### Fade Animations
- `fadeInUp`: Opacity 0→1, translateY 30px→0, 0.6s
- `fadeIn`: Opacity 0→1, 0.8s
- `pulse-glow`: Box-shadow pulse, 2s infinite
- `pulse-slow`: Opacity pulse, 4s infinite
- `line-pulse`: Opacity pulse for SVG lines, 3s infinite

#### Delay Classes
- `.delay-100` through `.delay-1200`
- Used for staggered text reveals

### Responsive Breakpoints
- **Mobile**: < 768px - Single column, reduced text
- **Tablet**: 768px - 1024px - 2-column features
- **Desktop**: > 1024px - Full 3-column layout

## Accessibility

### Reduced Motion Support
```css
@media (prefers-reduced-motion: reduce) {
  /* Disables float and pulse animations */
  /* Simplifies fade animations to 0.3s */
}
```

### Semantic HTML
- Proper heading hierarchy (h1 → h2 → h3)
- Section landmarks
- ARIA labels where needed

## Performance Optimizations

1. **CSS Animations Only**: No JavaScript animation libraries for background
2. **GPU Acceleration**: Uses `transform` and `opacity` only
3. **Intersection Observer**: Lazy-triggers animations on scroll
4. **Debounced Events**: Efficient scroll handling
5. **Static Generation**: Pre-rendered at build time

## Usage

### Development
```bash
npm run dev
```

### Production Build
```bash
npm run build

```

## Customization

### Changing Colors
Edit the color values in:
- `app/landing/components/AnimatedNodeGraph.tsx` (node colors)
- `app/landing/components/InteractiveCanvasPreview.tsx` (demo nodes)

### Adjusting Animation Speed
Modify animation durations in `app/globals.css`:
```css
.animate-float-1 { animation: float-1 12s ease-in-out infinite; }
/* Change 12s to desired duration */
```

### Adding More Features

```typescript
{
  icon: <YourIcon className="w-8 h-8" />,
  title: 'Feature Title',
  description: 'Feature description',
  gradient: 'from-color-500/20 to-color-600/20'
}
```

[Body]
Most teams have a decent understanding of their own codebase. The problem is that understanding lives in people’s heads, not anywhere a new teammate or an AI agent can actually access.

Documentation exists, but it’s flat, scattered, and usually out of date. Nobody reads it, and nobody updates it unless needed. And when you bring an AI into your workflow, it’s working from the same disconnected fragments everyone else is ignoring.

That’s why with the help of IBM Bob, we built Ingestalt.

Ingestalt is a spatial graph documentation engine that transforms your existing markdown files into an navigable interactive spatial canvas.

You point it at a YAML markdown, and your documentation becomes nodes on an infinite canvas, with relationships you can draw between them, schemas that define what each node type means, and an editor for keeping content current and updated. 

The map isn’t separate from your repo, so edits sync back to the markdown files automatically, thus it stays alive rather than becoming another process to focus more energy on.

The way Ingestalt handles different types of information is flexible by design. Instead of the tool deciding what database looks like, you define it yourself! 

Inside the tool, each definition lives in your workplace alongside everything else. This means your project’s structure reflects how your team actually thinks about it, not the tool that thinks you should organize things. And because everything syncs back to your files, anyone new coming to the codebase has somewhere real to start!
